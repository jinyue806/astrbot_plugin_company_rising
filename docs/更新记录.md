## 2026-06-09 · 校园筹备期与阶段边界修复

### 背景

用户反馈 `/startgame` 仍像“大学生创业/CEO/初始资金”开局。随后按 grill-me 方式做严格审查，发现问题不止文案：校园期和公司期共用状态结构，部分公司期字段和指令会污染校园存档。

### 修复

- **校园期身份口径修正**：`/startgame` 统一改为“校园筹备期”。校园期明确没有公司、没有 CEO、没有 AP、没有公司启动资金；`/状态`、`/面板`、`/真状态` 在校园期只显示积蓄、声望、人脉、技能和毕业倒计时。
- **运行时配置不再污染校园 AP**：`_sync_runtime_config()` 原先无条件把 `ceo.max_ap/ap` 写成配置默认值，导致校园存档虽然不显示 AP，但事实字段仍是 3。现在 `phase="campus"` 时强制保持 `ceo.ap/max_ap=0`，并清空 CEO 特质。
- **校园转公司写入顶层状态**：`campus_to_company()` 原先只写 `state["company_state"]`，顶层仍保留校园/默认字段。现在转换后顶层状态即为公司状态，同时保留校园快照。
- **启动资金来源修正**：校园转公司启动资金只来自校园期真实积蓄；有投资人时额外加投资款并记录股权稀释。不再使用硬编码“兼职/家里/学长投资”作为凭空资金来源。
- **行业归一修正**：`/创业 <公司名> <行业>` 找到行业后存行业名，不再把 `"1"` 这类编号写入 `meta.industry`。修复后行业权重、DBS 风险种子、融资 PE 估值恢复正常。
- **48 个月毕业结局接入**：校园动作推进月份后会检查 `campus.months_played`。达到 48 个月后按校园声望和隐藏背景生成结局、归档并清档。
- **校园结局背景映射修复**：`campus_endings.get_ending(win=False)` 原先用“普通/中产/富二代”直接查 `FAILURE_ENDINGS`，导致中产/富二代失败也拿普通家庭文案。现映射为 `real_unemployment` / `exam_grind` / `inherit_fortune`。
- **校园事件门禁修复**：`/选 <事件ID> <编号>` 原先从全量事件库查事件，玩家可在 0 个月直接选择黑客松等奖励事件。现在只允许选择当前可用事件。
- **公司期指令阶段挡板**：校园期使用 `/研发`、`/招聘`、`/录用`、`/跳过`、`/融资`、`/IPO`、`/成就`、`/天赋`、`/历史`、`/换办公室`、`/团队`、`/忠诚度`、`/离职`、`/辞退`、`/技能`、`/咨询`、`/KPI`、`/加薪`、`/考勤`、`/设施`、`/记忆`、`/谁` 时，会提示先完成校园期并注册公司，不再进入公司逻辑或写坏存档。
- **`/跳过` 空档污染修复**：未开局时 `/跳过` 不再创建默认公司存档。
- **月份计算修正**：`parse_time_to_month_idx("1年1月")` 从 13 修为 1；`1年12月=12`，`3年12月=36`。修复开局直接解锁“撑过 12 月”和结局回顾时长虚高一年。
- **帮助与交互提示修正**：`/帮助` 区分校园期和公司期；`/相亲` 不再提示“回复数字”，改为提示 `/选 事件ID 编号`。

### 回归覆盖

- 新增/更新校园期回归：校园状态不显示 CEO/AP/默认 50 万，校园转公司现金来自积蓄，行业编号归一为行业名。
- 新增事件门禁回归：0 个月不能直接选择黑客松。
- 新增校园失败结局回归：普通/中产/富二代分别映射到对应结局。
- 更新成就和归档时长回归：`1年1月` 不解锁 survivor，`1年12月` 解锁 survivor，`3年12月` 解锁 veteran。

### 修复

- **员工名册断链**：`recruit()` 原先只更新 `staff` 聚合计数，没有把录用候选人写入 `state["employees"]`。后果是 `/团队` 显示暂无员工，KPI、加薪、忠诚度、员工活动、离职系统都找不到新员工。已改为录用时写入员工对象，并返回 `employee` 供 `/录用` 初始化技能。

## 2026-06-07 · 全量开发记录

### 已修复的 Bug

**Bug 1: async/sync KV 失配** — `game_state.py`：`_load()`、`save()`、`delete()`、`has_game()` 调了异步 KV 接口但没加 `await`，数据从未写入持久化。修复：全部改成 `async def` + 加 `await`，`_get_gs()` 同步→异步 + 所有调用点加 `await`。影响：QQ 上之前玩的所有档都没存下来，修之后正常。

**Bug 2: `json.loads` 套 dict** — `get_kv_data()` 通过 SharedPreferences 返回的已经是解析好的 dict，`load()` 和 `has_game()` 又对 dict 调 `json.loads()`。修复：`load()` 直接 `self.state = raw`，`has_game()` 用 `raw.get("meta", {})` 替代。

**Bug 3: LLM 回复被截断** — SYSTEM_PROMPT 里设了"单次回复 ≤ 300 字"，月报内容多直接断半截。修复：删掉两处 300 字硬上限，改为由 `_chunk_text` 按 1500 字切块 yield。`_yield_chunked` 在 `async def` 中无法用 `yield from`，改回普通方法 `_chunk_text` 返回 `list[str]`。

**Bug 4: `dict(DEFAULT_STATE)` 浅拷贝污染模块默认值** — `GameState.__init__` 和 `reset()` 都用 `dict(DEFAULT_STATE)`，子 dict 仍与 DEFAULT_STATE 共享引用。后果：玩家写入 `state["meta"]["X"] = Y` 污染模块级 DEFAULT_STATE，重置后带脏数据。修复：两处都改 `copy.deepcopy(DEFAULT_STATE)`。教训：mutable default 永远用 deepcopy。

**Bug 5: `Context` 对象没有 `get_kv_data`** — AstrBot 新版 (>= 4.9.2) 把 KV 方法移到了 PluginKVStoreMixin。永久方案：改用 file-based 存储（`data/plugin_data/astrbot_plugin_company_rising/{user_id}.json`），完全脱离 framework KV API。教训：plugin 自家数据用 file，framework KV 留给 framework 状态。

**Bug 6: PowerShell 中文 mojibake** — PowerShell code page 936 + `$OutputEncoding` US-ASCII。临时修：chcp 65001 + 显式 UTF-8。永久固化：`bench/run_unicode.ps1`（4 target）。注意：≥ 符号在某些终端仍是字体问题，非编码问题。
