# 更新日志

本文件面向 AstrBot 插件页面展示，只保留用户需要看到的短日志。完整开发记录见 `更新记录.md`。

## [Unreleased]

### 新增
- 场景参数面板：`/公司 llm` 查看/调节每个场景的 temperature 和 max_tokens（存储于存档，持久化）
- `/llm 设置 <场景> <参数>=<值>` 运行时覆写 LLM 参数
- `/llm 重置 [场景]` 恢复默认

### 改进
- `prompt_templates.py` 结构化模板系统：PromptTemplate 数据类 + 注册表 + 按场景绑定参数
- `llm_service.py` LLM 调用服务：provider 解析、去重、场景参数三层回退（覆写 > 默认 > 全局配置）
- `_llm_chat()` 保持向后兼容，内部委托 LLMService

## [v1.7.0] - 2026-06-09
- 修复校园期误用公司期指令的问题；`/研发`、`/招聘`、`/融资`、`/IPO`、`/成就`、员工管理、设施等指令会提示先注册公司。
- 修复校园转公司后的行业数据：行业编号会归一为行业名，后续行业权重、融资估值、风险事件恢复正常。
- 修复校园 48 个月毕业结局未接入、校园失败结局背景映射错误的问题。
- 修复 `/选` 可绕过校园事件条件的问题，不能再提前触发黑客松等未解锁事件。
- 修复时间月份计算偏移，避免新公司开局直接解锁“撑过 12 月”，结局回顾时长也不再多算一年。
- 修复 `/跳过` 在未开局时创建默认存档的问题。

### 调整
- 可见指令收敛为 `/公司`、`/校园`、`/帮助`、`/真状态`；公司期和校园期动作改走子命令，旧直达写法通过兼容路由继续可用。
- `/帮助` 区分校园期与公司期，`/相亲` 的选择提示改为真实可用的 `/选 事件ID 编号`。
- 撤销 `/cr`、`/公司崛起` 等额外兼容入口，避免命令面继续膨胀；保留旧直达写法的兼容路由。
- 压缩页面更新日志，详细技术记录移入 `更新记录.md`。
- `/重置` 改为普通玩家可用，方便玩家自行清档重开；`/真状态` 仍保留管理员权限。

### 工程
- 新增 `storage.py`，统一插件数据目录解析，保持原存档目录不变。
- 新增 `docs/best-practices.md`，沉淀其他 AstrBot 插件的可复用工程实践。
- 新增通用参考文档 `data/plugins/PLUGIN_BEST_PRACTICES.md`，供后续插件开发查看。
- 补充校园期状态、事件门禁、行业归一、毕业结局、背景结局、月份计算等回归测试。

## v1.7.0 · 游戏深度增强 (2026-06-08)

### 新增
- 天赋消费端补全：研发、融资、客户、员工、办公室、IPO 等天赋效果落地。

- 修复 `/KPI <员工> <指标> <目标>` 按文档调用时误报用法的问题。
- 修复 Skill 文档编码和状态 schema 与代码不一致的问题。

### 验证
