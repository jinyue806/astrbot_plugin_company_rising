# 紧凑入口速查 (v1.7.0)

后台可见命令已收敛为 4 个：`/公司`、`/校园`、`/帮助`、`/真状态`。
旧直达写法仍兼容，例如 `/启动游戏`、`/研发`、`/招聘`、`/下一月`、`/startgame`。

## 开局入口

| 指令 | 触发 | 说明 |
|---|---|---|
| `/启动游戏 <公司> <行业编号> <特长编号> [难度]` | 直接创业 | 兼容直达入口，不再作为后台可见命令注册 |
| `/校园 开始` | 校园筹备期 | 48 个月学生阶段，积累后 `/校园 创业` 注册公司 |
| `/帮助` | 指令列表 | 显示紧凑入口 |

## 公司期入口

统一使用 `/公司 <子命令>`：

| 子命令 | 触发 | AP / 前置 |
|---|---|---|
| `研发 <项目名>` | 立项 | -2 AP, 现金>=10万 |
| `项目取消 <项目名>` | 取消项目 | -1 AP, 进度<80% |
| `招聘 [研发|设计|营销]` | 展示候选人 | AP>=1 |
| `录用 <编号>` | 录用候选人 | -1 AP, 有待录用候选人 |
| `跳过` | 跳过招聘 | 有待录用候选人 |
| `下一月` | 推进时间和月报 | 有存档 |
| `状态` | 一行战报 | 有存档 |
| `面板 [财务|团队|项目|竞争对手|ceo]` | 详细面板 | 有存档 |
| `融资` | 种子轮融资 | -1 AP, 声誉>=30, 融资冷却结束 |
| `IPO` | 公司上市 | -1 AP, 现金/声誉/团队/项目达标 |
| `重置` | 删除存档 | 有存档 |

## 低频公司管理

这些也走 `/公司 <子命令>`：

| 子命令 | 触发 | 说明 |
|---|---|---|
| `办公室 [A|B|C|D]` | 升级或降级办公室 | -1 AP + 装修费 |
| `咨询 [问题]` | 付费战略咨询 | -1 AP + 2万 |
| `设施 查看` / `设施 购买 <名称>` | 查看或建设设施 | 现金和办公室档位满足 |
| `成就` / `天赋 [天赋ID]` | 成就与 CEO 天赋 | 天赋解锁消耗天赋点 |
| `历史 [N]` / `记忆` / `谁 <员工名>` | 公司历史和图谱档案 | ontology 可用时信息更完整 |
| `团队` / `忠诚度` / `技能 <员工名>` | 员工信息 | 有员工 |
| `离职 <员工> <加薪|威胁|接受>` | 离职谈判 | 有离职风险 |
| `辞退 <员工名>` | 辞退员工 | 支付赔偿，团队士气下降 |
| `KPI <员工> <指标> <目标>` / `KPI 取消 <员工>` | KPI 管理 | 指标: progress / loyalty / skill_avg |
| `加薪 <员工> <金额|同意|拒绝|谈判>` | 主动加薪或处理要求 | 扣现金，影响忠诚度 |
| `考勤` | 本月活动摘要 | 月报后查看 |
| `结局录` / `回顾 [序号]` | 历史结局 | 有归档时可回顾 |

## 校园期入口

统一使用 `/校园 <子命令>`：

| 子命令 | 触发 | 说明 |
|---|---|---|
| `开始` | 启动校园模式 | 48 个月校园筹备期 |
| `打工 [tutor|freelance|intern]` | 赚钱 | 增加积蓄和少量声望 |
| `做产品 <产品名>` | 做产品 | 成功增加校园声望 |
| `比赛` | 参加比赛 | 黑客松等比赛，可能得奖金 |
| `社交` | 拓展人脉 | 可能找到合伙人 |
| `找合伙人` | 寻找合伙人 | 提高晋升机会 |
| `相亲` | 相亲事件 | 可能遇到投资人或富二代 |
| `选 <事件ID> <选项号>` | 选择事件 | 处理校园事件选项 |
| `创业 <公司名> <行业>` | 注册公司 | 达到晋升条件后进入公司期 |

## `/下一月` 完整流程

1. `advance_month()` 时间 +1，扣 fixed / rent / salary。
2. 规模管理成本：超过 3 名员工后，每人每月 +0.5 万。
3. 项目进度按产能推进，并应用 `dev_speed_bonus`。
4. 技术突破天赋可让项目直接完成。
5. 项目收入边际递减：每多完成 1 个 -8%，最低 30%。
6. 声誉软上限：超过 50 的部分每月衰减 5%。
7. `roll_monthly_event()` 触发随机事件，受 `event_frequency` 控制。
8. `advance_competitors()` 推进竞争对手，受 `competitor_enabled` 控制。
9. 员工忠诚、技能、自动离职和员工活动结算。
10. `evaluate_kpis()` 处理 KPI，受 `kpi_enabled` 控制。
11. `apply_facilities()` 扣设施维护费并应用设施效果。
12. AP 重置为 `max_ap + ap_bonus`。
13. cash < 0 时进入结局判定，危机公关天赋可能自动化解。
14. `build_monthly_prompt()` 注入真实状态、事件、对手、员工和 KPI。
15. `_llm_chat()` 使用 CORE + MONTHLY prompt 生成月报。
16. 推送顺序：里程碑 -> 跑路警告 -> GM 月报 -> 数据事实 -> 事件 -> 对手 -> 员工 -> KPI -> 加薪要求。

## CEO 天赋树

| 分支 | 1 级 | 2 级 | 3 级 |
|:---|:---|:---|:---|
| 技术 | 敏捷研发：研发速度 +15%，立项成本降低 | 技术突破：完成声誉增加，概率直接完成 | 系统架构师：产能增加，并行项目 +1 |
| 商业 | 融资大师：估值增加，稀释降低，冷却缩短 | 市场洞察：客户增长和营销效率增加 | 谈判大师：IPO 估值增加，办公室升级折扣 |
| 管理 | 人心所向：员工流失降低，忠诚下限提高 | 危机公关：负面事件减免，可能自动解危 | 成本控制师：固定成本、租金、薪资压力降低 |

## 员工技能

| 岗位 | 技能 |
|:---|:---|
| tech | 编程、架构、测试、运维、安全 |
| design | 界面、体验、调研、原型、无障碍 |
| marketing | 销售、SEO、品牌、内容、分析 |

- 自然成长：每月 +0.1 到 +0.15。
- 员工状态以 `state["employees"]` 为准，`staff` 只负责人数和薪资聚合。
